üöÄ Overview

The K PLUS Deep Link Handler is a JavaScript library designed to seamlessly open the K PLUS mobile banking application with authentication tokens. It provides intelligent fallback mechanisms and device-specific handling for optimal user experience across different platforms and browsers.

üéØ Compatibility

  • ‚úÖ Safari iOS 15+
  • ‚úÖ Android 10+ browsers
  • ‚úÖ Huawei devices with HMS
  • ‚úÖ Social media in-app browsers (Line, Messenger, Facebook)
üì¶ Installation & Usage
Include in HTML:
<script src="kplus-deeplink.js"></script>
Or import in Node.js:
const { KPlusDeepLinkHandler, kplusHandler } = require('./kplus-deeplink.js');

üåç Global Instance Available

A global instance kplusHandler is automatically created and available for immediate use.

üèóÔ∏è Constructor
new KPlusDeepLinkHandler() Constructor
Creates a new instance of the K PLUS Deep Link Handler with predefined configuration for different device types and environments.

üìù Properties Initialized:

huaweiUrl string
Huawei App Gallery deep link URL with placeholder tokens
generalUrl string
General fallback URL for web browsers
fallbackDuration number
Duration to wait before fallback (3000ms)
scheme string
URL scheme for K PLUS app ('kbank.kplus://')
socialApps array
Array of social app identifiers ['line', 'messenger', 'fban', 'fbav']
Example:
const handler = new KPlusDeepLinkHandler();
‚öôÔ∏è Methods
isAndroidDevice() Method
Detects if the current device is running Android by analyzing the user agent string.

‚Ü©Ô∏è Returns:

boolean - true if Android device, false otherwise
Example:
const isAndroid = kplusHandler.isAndroidDevice(); console.log(isAndroid); // true or false
isHuaweiDevice() Method
Detects if the current device is a Huawei device by checking for Huawei, Honor, or HMS identifiers in the user agent.

‚Ü©Ô∏è Returns:

boolean - true if Huawei device, false otherwise
Example:
const isHuawei = kplusHandler.isHuaweiDevice(); if (isHuawei) { console.log('Using Huawei-specific deep link'); }
isSocialApp() Method
Detects if the current browser is running inside a social media application (Line, Messenger, Facebook, etc.).

‚Ü©Ô∏è Returns:

boolean - true if social media app, false otherwise
Example:
const isSocial = kplusHandler.isSocialApp(); if (isSocial) { console.log('Running inside social media app'); }
parseQueryParams(queryString) Method
Parses a query string into an object with decoded parameter names and values.

üì• Parameters:

queryString string required
Query string to parse (e.g., "?nextAction=authenwithkplus&tokenId=xxxx")

‚Ü©Ô∏è Returns:

Object - Object containing parsed parameters with decoded keys and values
Example:
const params = kplusHandler.parseQueryParams("?nextAction=authenwithkplus&tokenId=abc123"); console.log(params); // Output: { nextAction: "authenwithkplus", tokenId: "abc123" }
buildQueryString(params) Method
Builds a URL-encoded query string from a parameters object.

üì• Parameters:

params Object required
Parameters object to convert to query string

‚Ü©Ô∏è Returns:

string - URL-encoded query string (without leading '?')
Example:
const queryString = kplusHandler.buildQueryString({ nextAction: "authenwithkplus", tokenId: "abc123", userId: "user@example.com" }); console.log(queryString); // Output: "nextAction=authenwithkplus&tokenId=abc123&userId=user%40example.com"
openKPlusApp(queryParams) Method
Opens the K PLUS app with the provided authentication parameters. Automatically detects the device type and browser environment to choose the optimal opening method.

üì• Parameters:

queryParams string required
Query parameters string containing at least tokenId and nextAction

‚ö†Ô∏è Throws:

Error - "QueryParams is required" if queryParams is null/undefined
Error - "tokenId parameter is required in queryParams" if tokenId is missing
Error - "nextAction parameter is required in queryParams" if nextAction is missing
Example:
// Basic usage kplusHandler.openKPlusApp("?nextAction=authenwithkplus&tokenId=abc123"); // With additional parameters kplusHandler.openKPlusApp("?nextAction=authenwithkplus&tokenId=abc123&userId=user123"); // Error handling try { kplusHandler.openKPlusApp("?nextAction=authenwithkplus&tokenId=abc123"); } catch (error) { console.error('Failed to open K PLUS app:', error.message); }

üîÑ Behavior by Platform:

  • Social Apps + Android/Huawei: Uses URL scheme with fallback mechanism
  • Huawei Devices: Uses Huawei App Gallery deep link
  • Other Devices: Uses general web URL with query parameters
üåê Global Functions
window.openKPlus(queryParams, displayMode) Global Function
Global convenience function for opening K PLUS app with optional display mode selection (popup modal or direct navigation).

üì• Parameters:

queryParams string required
Query parameters string containing authentication tokens
displayMode string optional
Display mode: 'POPUP' for modal confirmation, 'DIRECT' for immediate navigation (default)
Example:
// Direct navigation (default) openKPlus("?nextAction=authenwithkplus&tokenId=abc123"); // With popup confirmation openKPlus("?nextAction=authenwithkplus&tokenId=abc123", KPlusDisplayMode.POPUP); // Using constants openKPlus("?nextAction=authenwithkplus&tokenId=abc123", window.KPlusDisplayMode.DIRECT);
createKPlusModal(queryParams) Function
Creates and displays a modal dialog asking for user confirmation before opening the K PLUS app. Provides a user-friendly interface with Thai language support.

üì• Parameters:

queryParams string required
Query parameters to pass to the K PLUS app when user confirms
Example:
// Show confirmation modal createKPlusModal("?nextAction=authenwithkplus&tokenId=abc123");

üìã Available Values:
